# Agent notes – Badminton Tournament Planner

## Running tests

Tests run inside the Docker container (requires access to the PostgreSQL database).

### Run all tests

```powershell
docker-compose run --rm --entrypoint python web -m pytest -v
```

### Run tests for a specific app

```powershell
docker-compose run --rm --entrypoint python web -m pytest players/tests.py -v
docker-compose run --rm --entrypoint python web -m pytest tournaments/tests.py -v
```

### Run a specific test class or function

```powershell
# Class
docker-compose run --rm --entrypoint python web -m pytest players/tests.py::PlayerBulkDeleteTest -v

# Single test
docker-compose run --rm --entrypoint python web -m pytest players/tests.py::PlayerBulkDeleteTest::test_delete_selected_players -v
```

### Notes

- `--entrypoint python` bypasses `entrypoint.sh` (which would otherwise start gunicorn).
- The `db` container is started automatically via `depends_on` — it does not need to be running beforehand.
- Django creates a separate test database (`test_badminton`) and drops it afterwards.
- Configuration: `pytest.ini` → `DJANGO_SETTINGS_MODULE = tournament_planner.settings`

## Localisation (i18n)

The project supports **English (default)** and **Danish**. All user-facing text must be translated.

### Rules for new code

1. **Wrap all user-facing strings** with `_()`:
   ```python
   from django.utils.translation import gettext as _
   from django.utils.translation import gettext_lazy as _lazy  # use in models/forms

   messages.success(request, _("Player deleted."))
   verbose_name = _lazy("Player")
   ```

2. **Add strings to the `.po` file** every time new text is added in views, models or forms.  
   File location: `locale/da/LC_MESSAGES/django.po`

3. **Format for a new translation entry in `django.po`**:
   ```po
   #: players/views.py
   msgid "The English source string exactly as written in the code."
   msgstr "The Danish translation."
   ```
   For format strings with variables:
   ```po
   #: players/views.py
   #, python-format
   msgid "%(n)s player(s) deleted."
   msgstr "%(n)s spiller(e) slettet."
   ```

4. **Compile `.po` → `.mo`** so Django can use the translations:
   ```powershell
   # Inside a running container:
   docker-compose exec web python manage.py compilemessages

   # Or via docker-compose run (requires image rebuild to take effect in production):
   docker-compose run --rm --entrypoint python web manage.py compilemessages
   ```
   > In production, compilation happens automatically during `docker-compose build` (see `Dockerfile`).

### Files and configuration

| File/folder | Purpose |
|---|---|
| `locale/da/LC_MESSAGES/django.po` | Source for Danish translations — **edit this manually** |
| `locale/da/LC_MESSAGES/django.mo` | Compiled binary — **do not edit**, generated by `compilemessages` |
| `tournament_planner/settings.py` | `LANGUAGES`, `LOCALE_PATHS`, `USE_I18N = True` |
| `Dockerfile` | Runs `compilemessages` during build so `.mo` is always up to date |

### Workflow for changes

```
New/changed string in code
        ↓
Add msgid + msgstr in locale/da/LC_MESSAGES/django.po
        ↓
docker-compose build web   ← compilemessages runs automatically
        ↓
docker-compose up
```
